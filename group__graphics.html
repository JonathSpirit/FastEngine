FGE_API std::unique_lock< std::mutex > fge::texture::AcquireLock ( )

Acquire a unique lock, with the texture manager mutex.

In order to use iterators, you have to acquire a unique lock from this function. The lock is not differed and will lock the mutex.

Returns
A unique lock bound to this mutex

◆ IteratorBegin()

FGE_API fge::texture::TextureDataType::const_iterator fge::texture::IteratorBegin ( const std::unique_lock< std::mutex > &  lock)

Get the begin iterator of the texture manager.

You have to provide a valid reference to a unique lock acquire with the function AcquireLock(). This function will throw if one of this is not respected :

  • The lock does not owned the associated mutex.
  • The mutex pointer of the lock does not correspond to this mutex.
Parameters
lockA unique lock bound to this mutex
Returns
The begin iterator of the texture manager
